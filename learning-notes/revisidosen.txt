Tujuan file:
File ini dipakai untuk tracking revisi dosen agar setiap poin bisa dipantau

Status yang dipakai:

- SUDAH = sudah tersedia dan bisa dipakai

====================================================================
RINGKASAN KONTEKS SISTEM SAAT INI
====================================================================

Arsitektur utama:
- Frontend React
- Backend Express
- AI Engine Flask + PuLP

Alur data saat ini:
User input bahan di frontend
-> frontend hitung preview total gizi
-> frontend kirim payload ke backend
-> backend validasi request
-> backend forward ke AI engine
-> AI engine optimasi kombinasi menu
-> hasil dikirim lagi ke frontend

Catatan penting:
- Sistem saat ini masih berorientasi pada optimasi per porsi, bukan total kebutuhan
  untuk sejumlah siswa.
- Sistem saat ini masih meminta input minimum kalori dan minimum protein secara
  manual pada dashboard.
- Database MongoDB sudah mulai dipasang di backend, tetapi alur CRUD dan integrasi
  frontend belum selesai.
